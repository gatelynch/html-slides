# HTML 簡報頁型

每頁使用 `<section class="slide">`。如果需要講者備註，可以在 section 上加入 `data-note`，或在內部使用隱藏的備註區塊。

```html
<section class="slide" data-note="這裡放講者提示">
  <div class="slide-inner">
    <p class="eyebrow">章節標籤</p>
    <h2>頁面標題</h2>
    <p class="lead">核心訊息。</p>
  </div>
</section>
```

## 基本 HTML 架構

- `html` 使用垂直 scroll snap。
- `.slide` 是完整 viewport，高度固定為 `100vh` / `100dvh`，不可依賴內部捲動。
- `.slide-inner` 控制最大寬度與內容節奏。
- 導覽、進度條與動畫可加入，但可依簡報場合省略。
- 顏色、字體、表面、邊框都透過 style token 控制。

## 12 種通用版型

### 1. Cover

開場頁。用於標題、場次、核心承諾。

- 必含：大標題、短副標。
- 可含：日期、講者、單一主視覺。
- 避免：議程、長段落、多個 logo 搶焦點。

### 2. Agenda

讓觀眾知道接下來的路徑。

- 必含：2 到 4 個段落。
- 可含：目前段落標記。
- 避免：超過 4 點，變成索引清單。

### 3. Two-column

左文右圖或左右對照，是最常用的內容頁。

- 必含：一側核心說明，一側視覺證據。
- 可含：短 checklist、截圖、流程圖、案例素材。
- 避免：左右兩邊都塞滿文字。

### 4. Stat

用 1 到 3 個大數字建立重量感。

- 必含：數字、單位、語意標籤。
- 可含：短補充、來源。
- 避免：沒有單位或數字意義不明。

### 5. Feature Grid

用 3 到 6 張卡片整理能力、觀點或模組。

- 必含：每張卡片 1 個標題 + 1 句說明。
- 可含：小圖示、代表性範例。
- 避免：卡片文字長短失衡、超過 6 張。

### 6. Comparison

用表格或雙欄比較兩種選項。

- 必含：比較維度。
- 可含：推薦欄、勝出標記。
- 避免：兩堆鬆散條列，沒有共同標準。

### 7. Process

呈現 3 到 5 個步驟。

- 必含：步驟名稱、短描述、方向關係。
- 可含：簡潔圖示或編號。
- 避免：每步超過兩行，或步驟關係不清楚。

### 8. Timeline

呈現時間、里程碑、路線圖。

- 必含：時間點、事件、目前位置或下一步。
- 可含：已完成 / 進行中 / 待辦狀態。
- 避免：把時間軸做成普通 bullet list。

### 9. Chart

用圖表支撐定量論點。

- 必含：圖表標題、單位、資料來源或說明。
- 可含：堆疊長條、折線、圓環、分組長條。
- 避免：彩虹配色、超過 5 個類別、讓觀眾自己猜結論。

### 10. Code

用於程式碼、指令、prompt、設定片段。

- 必含：短片段、語境、要觀察的行。
- 可含：輸入 / 輸出對照。
- 避免：整頁小字程式碼。

### 11. Quote

用一句話建立觀點、情緒或權威。

- 必含：引言與來源。
- 可含：人物照片、組織 logo、案例背景。
- 避免：超過 3 行的長引文。

### 12. Closing

收束與下一步。

- 必含：一句收束語、下一步行動。
- 可含：QR code、連結、聯絡方式。
- 避免：把所有參考資料堆到最後一頁。

## 10 種教學版型

教學版型可和通用版型混用；它們的重點是讓學習者知道「現在要做什麼」。以下是 HTML 結構速查；每種頁型的設計規格與常見錯誤見 `teaching-patterns.md`。

### 13. Activity

完整活動任務頁。

- 必含：情境、任務、時間、產出、討論問題。
- 適合：分組討論、個人練習、課堂任務。

### 14. Try First

先讓學習者嘗試，不先給答案。

- 必含：清楚輸入、限制、提交形式。
- 適合：訓練判斷力、寫作、prompt、設計題。

### 15. Reveal

嘗試後揭示教師版本或範例版本。

- 必含：揭示內容、為什麼有效、可改進處。
- 適合：接在 Try First 後面。

### 16. Prompt Lab

針對 prompt 或 AI 工具操作的實驗頁。

- 必含：目標、prompt、變因、觀察點。
- 適合：AI 教學、工具實作、提示詞比較。

### 17. Compare Tools

比較不同工具、模型或方法。

- 必含：共同任務、比較維度、建議使用情境。
- 適合：工具選型、方法判斷。

### 18. Rubric

評分規準或品質階梯。

- 必含：維度、等級、可觀察行為。
- 適合：作品評量、回饋、學生自評。

### 19. Classroom Case

真實或擬真課堂案例。

- 必含：角色、情境、衝突或決策點。
- 適合：教師研習、班級經營、情境判斷。

### 20. Reflection

收斂學習者的觀察與下一步。

- 必含：反思問題、個人輸出、分享方式。
- 適合：活動後、章節收束、課程結尾。

### 21. Workflow

教一套可重複操作的流程。

- 必含：輸入、步驟、輸出、檢查點。
- 適合：AI 工作流、教案設計、資料處理。

### 22. Before/After

呈現修改前後差異。

- 必含：修改前、修改後、改動理由。
- 適合：寫作修訂、prompt 改寫、作品優化。

## 導覽與講者備註

基本 HTML 簡報可包含：

- 鍵盤左右鍵或上下鍵換頁。
- 空白鍵前進。
- 頁面進度條。
- 目前頁碼或低調導覽點。

換頁捲動有兩種模式，由 `<body>` 的 class 控制：

- `layout-horizontal`（預設）：傳統左右橫向換頁。
- `layout-vertical`：上下垂直捲動。

預設用橫向；若簡報偏長文、講義式或希望像網頁那樣滑動，可改成垂直。範本同時支援按 `V` 即時切換，或用網址參數 `?layout=vertical` / `?layout=horizontal` 覆寫,方便使用者依場合自選。

導覽元件要低調，不要搶走內容焦點。講者備註適合放開場句、提問、活動時間提醒、範例揭示前後的引導語。不要把講者備註預設成手機版功能；是否顯示、如何顯示，依該份簡報需求決定。
