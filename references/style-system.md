# 視覺風格系統

HTML 簡報的目標是「站在台上能被快速理解」，不是把文章縮進投影片。每頁應該有清楚視覺焦點、足夠留白、一致語氣，以及符合場合的資訊密度。

## 共同規格

所有風格都使用同一組 CSS token，讓版型可以跨風格重用：

```css
:root {
  --bg-primary: #ffffff;
  --text-primary: #111827;
  --text-secondary: #64748b;
  --accent: #2563eb;
  --accent-dim: rgba(37, 99, 235, 0.10);
  --surface: #f8fafc;
  --border: #dbe3ef;
  --on-accent: #ffffff;
  --font-title: -apple-system, BlinkMacSystemFont, "Noto Sans TC", "PingFang TC", "Microsoft JhengHei", sans-serif;
  --font-body: -apple-system, BlinkMacSystemFont, "Noto Sans TC", "PingFang TC", "Microsoft JhengHei", sans-serif;
}
```

- 版型 CSS 優先使用 token，不要把顏色硬寫死在單一版型裡。
- 一份簡報只選一種主風格；必要時可做少數深色封面或案例頁，但不要混成多套品牌。
- 卡片、表格、工具面板的圓角以 8px 左右為準，避免過度可愛或像套版。
- 強調色要節制；一頁通常只需要一個主要強調點。

## 六種風格

### Clean（預設）

清楚、正式、通用。適合研習、課程講義、產品說明、報告與大多數簡報。

- 背景：白色或極淺灰。
- 文字：深灰黑，輔助文字用冷灰。
- 強調：藍或藍綠，僅用在重點數字、行動按鈕、目前步驟。
- 氣質：安靜、精準、可掃讀。
- 避免：過多裝飾、大片漸層、每頁都做英雄式標題。

### Workshop

現場活動感較強，適合工作坊、實作課、分組討論。

- 背景：淺暖色或白底搭配溫和色塊。
- 文字：深灰黑。
- 強調：橘紅、綠或藍綠，用於任務、時間、產出。
- 氣質：明確、可操作、有節奏。
- 避免：只寫概念不寫任務；活動頁一定要有輸入、任務、產出。

### Notebook

像整理過的筆記與教案，適合知識梳理、閱讀導讀、反思活動。

- 背景：白色或很淡的紙感底色。
- 文字：深墨色。
- 強調：深藍、墨綠或焦點色。
- 氣質：沉穩、留白、帶有註記感。
- 避免：做成雜亂手帳；註記元素只服務結構。

### Story

以情境、照片、案例推進，適合故事型演講、課堂案例、前後轉變。

- 背景：依照片或案例素材決定，可用深色覆層提升可讀性。
- 文字：大而短，讓媒體承擔情緒與脈絡。
- 強調：單一高對比色，用於關鍵句、章節轉折。
- 氣質：敘事、沉浸、少字。
- 避免：照片只是裝飾；每張照片都要有敘事功能。

### Lab

工具實驗、AI prompt、程式碼、流程測試。適合技術教學與工具比較。

- 背景：淺色工作台或深色面板皆可。
- 文字：清楚的 sans，程式碼或 prompt 可使用 monospace。
- 強調：青色、綠色或藍色，用於輸入、輸出、變數、狀態。
- 氣質：界面感、實驗感、可重複操作。
- 避免：把 prompt 塞成小字牆；每個 prompt lab 都要有目標與觀察點。

### Dark

高對比暗色，適合舞台、錄影、強情緒開場或技術 demo。

- 背景：黑或近黑。
- 文字：白或淺灰。
- 強調：黃、青或亮藍。
- 氣質：集中、俐落、戲劇性。
- 避免：低對比灰字、彩虹色圖表、連續太多密集表格。

## 字級與密度

這是預設尺度，可依場地與投影距離微調，但不要把文字縮成文件。

| 元素 | 建議大小 | 用法 |
|---|---|---|
| 封面標題 | `clamp(3.2rem, 7vw, 6.4rem)` | 1 個主標，最多 1 個短副標 |
| 內容頁標題 | `clamp(2.4rem, 5.2vw, 4.6rem)` | 一眼讀完，避免抽象口號 |
| 區塊 / 步驟標題 | `clamp(1.5rem, 2.6vw, 2.2rem)` | 卡片、流程、活動步驟 |
| 內文 | `clamp(1.08rem, 1.7vw, 1.45rem)` | 短句，最多 1 到 2 行 |
| 註解 / 標籤 | `clamp(.9rem, 1.2vw, 1.08rem)` | 時間、角色、來源、提示 |

內容密度上限：

| 頁型 | 上限 |
|---|---|
| 封面 | 1 個標題 + 1 個副標 |
| 一般內容 | 1 個標題 + 5 個以內重點 |
| 卡片網格 | 1 個標題 + 6 張卡片以內 |
| 活動 | 情境、任務、時間、產出、討論問題各 1 組 |
| 圖片 | 1 個標題 + 1 張主圖，圖片高度約 74vh 以內 |
| Quote | 1 句引言，最多 3 行 + 來源 |

## 版面規則

- 每頁只承載一個主要訊息。
- 內容頁要有明確起點：小標籤、標題、主視覺或任務卡。
- 文字、圖片、卡片和圖表都要有明確對齊關係。
- 不連續使用同一種版型超過兩頁。
- 留白不足時，優先拆頁，不要縮到難以閱讀。
- 身為講義也要可讀；身為投影也要夠大。

## 動畫與導覽

動畫與導覽是建議，不是硬性要求；可依簡報場合省略。

- 建議保留：進度條、鍵盤前後頁、目前頁進場狀態。
- 可省略：導覽圓點、頁碼、複雜連續動畫、裝飾性 loop。
- 動畫距離小、時間短，避免彈跳、旋轉、過度模糊。
- 一頁最多一個主要進場動作，其餘元素淡入即可。
- loop 動畫只放在小型插圖或裝飾元素，不放在文字。
- 圖片獨立成頁時，靜態或淡入即可。
- 一定支援 `prefers-reduced-motion`，讓動畫可被系統設定降到最低。

## 常見問題

- 裝飾線、漸層塊、玻璃擬態太多，容易讓簡報看起來像套版。
- 每頁都用一樣的左右兩欄，節奏會變平。
- 標題太抽象時，觀眾不知道這頁要看什麼。
- 活動頁若只列概念，現場會不知道要做什麼。
- 風格名稱很漂亮但沒有落到版面規則，最後還是會變成普通條列。
